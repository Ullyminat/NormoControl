# NormoControl: Система автоматизированного нормоконтроля

Система NormoControl предназначена для верификации научно-исследовательских и учебных работ на соответствие установленным стандартам оформления (ГОСТ, локальные нормативные акты). Платформа осуществляет структурный и семантический анализ документов формата DOCX, сопоставляя их с динамически настраиваемыми профилями правил.

## Ключевые функциональные возможности

### Для экспертов и преподавателей
*   **Конфигуратор стандартов**: Модульная система настройки правил, включающая параметры полей, межстрочных интервалов, иерархии заголовков, оформления таблиц, формул и библиографических списков.
*   **Интеллектуальный анализ**: Слой верификации на базе Gemini API для проведения экспертной оценки содержания и семантической корректности фрагментов документа.
*   **Импорт конфигураций**: Механизм автоматического извлечения параметров оформления из эталонных документов (.docx) для быстрого создания новых стандартов.

### Для пользователей и студентов
*   **Система верификации**: Загрузка документов через интерфейс drag-and-drop с последующим автоматизированным анализом нарушений.
*   **Интерактивная визуализация**: Просмотрщик PDF с координатной привязкой найденных нарушений к конкретным фрагментам текста.
*   **Учет и статистика**: Личный кабинет с историей проверок, динамикой оценок и детальными отчетами по каждому документу.

## Технологический стек

| Компонент | Технология | Описание |
| :--- | :--- | :--- |
| **Backend** | Go 1.21+ (Gin Framework) | Высокопроизводительный API, параллельная обработка документов. |
| **Frontend** | React 18 (Vite) | Одностраничное приложение (SPA) с компонентной архитектурой. |
| **База данных** | SQLite 3 | Реляционная СУБД для хранения метаданных и конфигураций. |
| **Анализ текста** | Gemini Pro / Gemma 3 | Экспертный слой оценки семантики и логической структуры. |
| **Конвертация** | LibreOffice (Headless) | Серверное преобразование DOCX в PDF для визуального контроля. |
| **Развертывание** | Docker & Ansible | Контейнеризация и автоматизация управления конфигурациями серверов. |

## Архитектура системы

### Структура проекта
```text
├── ansible/                 # Скрипты автоматизации развертывания
├── backend/                 # Серверная часть (Golang)
│   ├── internal/
│   │   ├── ai/              # Интеграция с LLM провайдерами
│   │   ├── checker/         # Ядро анализа структуры документа
│   │   ├── database/        # Слой доступа к данным (GORM)
│   │   └── handlers/        # Логика обработки HTTP-запросов
│   └── fonts/               # Системные шрифты (GOST Type B)
├── frontend/                # Клиентская часть (React)
│   ├── src/
│   │   ├── features/        # Функциональные модули (Dashboard, Teacher, Student)
│   │   └── components/      # Общие UI-компоненты
│   └── public/              # Статические ресурсы и шрифты
└── docker-compose.yml       # Описание оркестрации контейнеров
```

## Развертывание

### Локальное развертывание (Docker)
1.  Склонируйте репозиторий.
2.  Создайте файл `.env` в директории `backend/` на основе `example.env`, указав `GEMINI_API_KEY`.
3.  Разместите файл шрифта `GOST Type B.ttf` в директориях `backend/fonts/` и `frontend/public/fonts/`.
4.  Выполните команду:
    ```bash
    docker compose up -d --build
    ```

### Автоматизированное развертывание (Ansible)
Система поддерживает автоматический деплой на удаленные серверы через Ansible. Конфигурация находится в директории `ansible/`.
```bash
ansible-playbook -i inventory.ini deploy.yml
```

## Безопасность и производительность
*   **RBAC**: Разграничение прав доступа на уровне API (преподаватель/студент).
*   **Безопасность сессий**: Использование HTTP-only Cookies для хранения JWT-токенов.
*   **Оптимизация**: Минимизация нагрузки на сервер за счет предварительного анализа XML-структуры документа перед запуском тяжелых процессов конвертации.
*   **Типографика**: Изолированная подсистема рендеринга шрифтов обеспечивает идентичность отображения документа в браузере и при печати.
